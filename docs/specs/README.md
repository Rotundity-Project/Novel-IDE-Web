# Specs（功能规格）

本目录存放“可交付能力”的规格文件。每个 spec 都应能追溯到代码实现与验证方式。

## 命名规则
- 使用递增编号：`0001-<slug>.md`。
- slug 使用小写英文与短横线，例如：`0001-auth-rest.md`。

## 状态约定
- Draft / Reviewed / Accepted / Implemented / Deprecated（见 `docs/spec-kit/README.md`）。

## 最小内容要求
- 背景与目标
- 需求与约束
- 业务规则（不变量）
- 验收标准
- 验证方式
- 实现落点（Implemented 后必须补齐）
